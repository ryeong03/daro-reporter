## 1. 백엔드 (Node.js + Express + TypeScript)

**배포**: Railway — `daro-reporter-production.up.railway.app`

### API 엔드포인트

| 엔드포인트 | 기능 | 파일 | 상태 |
|---|---|---|---|
| `POST /users/register` | 농업인 등록 + 보호자 + 기준심박 계산 | `api/user.ts` | 완료 |
| `POST /health` | 심박·걸음수·GPS 수신 → 감지 엔진 호출 | `api/health.ts` | 완료 |
| `POST /alert` | 낙상/BT끊김/앱크래시 알림 수신 | `api/alert.ts` | 완료 |
| `POST /twilio/voice-response` | TwiML 음성 안내 + 녹음 지시 | `api/twilio-webhook.ts` | 완료 |
| `POST /twilio/recording` | 녹음 콜백 → STT → AI분류 → 분기 | `api/twilio-webhook.ts` | 완료 |
| `POST /twilio/recording-status` | 녹음 완료 상태 콜백 | `api/twilio-webhook.ts` | 완료 |
| `POST /twilio/voice-status` | 통화 상태(완료/실패) 콜백 | `api/twilio-webhook.ts` | 완료 |

### 핵심 모듈

| 모듈 | 파일 | 기능 요약 |
|---|---|---|
| 2단계 감지 엔진 | `detection/engine.ts` | 심박 상승/급락 감지 → 비활동 확인 → 미회복 판단 → AI콜 트리거 |
| 상태 머신 | `state/state-machine.ts` | normal → stage1 → stage2 → observing → alert 전이 관리 |
| 기준선 계산 | `detection/baseline.ts` | 과거 7일 심박 기반 개인별 기준심박 산출 |
| AI콜 트리거 | `ai-call/trigger.ts` | 감지 → Twilio 전화 발신 흐름 제어 |
| Twilio 발신 | `ai-call/twilio.ts` | makeCall() — 전화 발신 + TwiML URL 설정 |
| Clova STT | `ai-call/stt.ts` | 녹음 URL → 한국어 텍스트 변환 |
| Claude 분류 | `ai-call/classify.ts` | STT 텍스트 → safe/emergency/unclear 판단 |
| 긴급 알림 | `notify/emergency.ts` | Slack + SMS + 보건소 전화 일괄 발송 |
| Slack 알림 | `notify/slack.ts` | Block Kit 형식 Slack 메시지 전송 |
| SMS 발송 | `notify/sms.ts` | Twilio SMS로 보호자/보건소 문자 |
| 기상청 API | `external/weather.ts` | 초단기실황 → 체감온도 계산 → 감지 가중치 |
| 카카오맵 API | `external/kakao-map.ts` | GPS 좌표 → 주소 변환 (역지오코딩) |

### 최근 수정 사항

| 항목 | 변경 내용 |
|---|---|
| AI 분류 프롬프트 | safe/emergency/unclear 세부 판단 기준 + 참고 말뭉치 추가 (스펙 문서 기반) |
| TwiML 음성 멘트 | 어르신 이름 동적 삽입 + 스펙대로 문구 전면 교체 (safe/emergency/unclear 각각) |
| Supabase 에러 처리 | voice-response에서 .single() 쿼리 try/catch + error 체크 추가 |
| test-call.ts 위치 | backend/ 루트 → backend/scripts/로 이동 |

## 2. 데이터베이스 (Supabase PostgreSQL)

**스키마**: `backend/src/db/schema.sql`

| 테이블 | 용도 | 주요 컬럼 |
|---|---|---|
| `users` | 등록 농업인 | id, name, phone, device_id, baseline_bpm, gender, birth_date |
| `guardians` | 보호자 | user_id, name, phone, relation |
| `health_data` | 심박·걸음수·GPS | user_id, heart_rate[], steps_10min, location, detection_state |
| `alerts` | 이상/낙상 알림 | user_id, type, severity, location, resolved_at |
| `call_logs` | AI콜 기록 | user_id, call_sid, stt_text, classification, attempt |
| `notification_logs` | 알림 발송 기록 | alert_id, channel(slack/sms/call), status |

### 화면 구성

| 화면 | 파일 | 기능 |
|---|---|---|
| 온보딩 (3단계) | `OnboardingScreen.tsx` | 1. 기본정보 → 2. 보호자 등록 → 3. 워치 연결 |
| 홈 대시보드 | `HomeScreen.tsx` | 심박수 실시간 표시 + 상태카드 + 걸음수 + GPS + 연결상태 |

| 모듈 | 파일 | 기능 |
|---|---|---|

| GPS 위치 | `services/locationService.ts` | 정상 5분/이상 30초 주기 위치 수집 |

| 데이터 동기화 | `services/dataSync.ts` | 10분 주기 서버 전송 + 오프라인 큐잉 + 재전송 |
