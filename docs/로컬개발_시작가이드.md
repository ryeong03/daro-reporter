# 로컬 개발 시작 가이드

> 프로젝트를 처음 받아서 로컬에서 실행하기까지의 전체 과정.

---

## 사전 준비

| 도구 | 설치 확인 | 없으면 |
|---|---|---|
| Node.js (v18+) | `node -v` | https://nodejs.org |
| npm | `npm -v` | Node.js 설치 시 같이 옴 |
| Git | `git --version` | https://git-scm.com |

---

## 1단계: 프로젝트 클론

```bash
git clone https://github.com/ryeong03/daro-reporter.git
cd daro-reporter
```

---

## 2단계: 백엔드 세팅

### 패키지 설치

```bash
cd backend
npm install
```

### 환경변수 설정

```bash
cp .env.example .env
```

`.env` 파일을 열어서 아래 두 개만 채우면 기본 동작 가능:

```env
SUPABASE_URL=https://eecsoxllkddrvavhpgpx.supabase.co
SUPABASE_SERVICE_ROLE_KEY=(세령에게 받기)
```

> 나머지 키(Twilio, Claude 등)는 비워두면 해당 기능만 안 됨. 서버 실행은 됨.

### 서버 실행

```bash
npx ts-node src/index.ts
```

성공 시 출력:
```
[Hero] Server running on port 3000
```

### 확인

브라우저에서 http://localhost:3000 접속 →
```json
{"status":"ok","service":"Hero Backend"}
```
이 보이면 성공!

---

## 3단계: 대시보드(프론트) 세팅

**새 터미널 탭** (Cmd+T) 열어서:

```bash
cd dashboard
cp .env.example .env
npm install
npm start
```

`dashboard/.env` 에 **JavaScript 키** 설정 (REST 키와 다름):

```env
REACT_APP_API_URL=http://localhost:3000
REACT_APP_KAKAO_MAP_KEY=카카오_JavaScript_키
```

카카오 개발자 콘솔 → 플랫폼 → Web → `http://localhost:3000` 등록.

브라우저에 http://localhost:3000 (또는 백엔드와 포트가 겹치면 3001) 이 열립니다.

---

## 4단계: 동작 테스트

**또 다른 터미널 탭** (Cmd+T) 열어서:

### 사용자 등록
```bash
curl -X POST http://localhost:3000/users/register \
  -H "Content-Type: application/json" \
  -d '{"name":"김할머니","phone":"01011112222","device_id":"fit3-001"}'
```

성공 응답:
```json
{"user":{"id":"uuid-값","name":"김할머니","phone":"01011112222",...}}
```

→ 대시보드(localhost:3001) 새로고침하면 "김할머니"가 목록에 보임!

### 헬스 데이터 전송 (정상 심박)
```bash
curl -X POST http://localhost:3000/health \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "fit3-001",
    "user_id": "위에서_받은_UUID",
    "timestamp": "2026-05-25T15:00:00+09:00",
    "heart_rate": [{"t": "2026-05-25T15:00:00+09:00", "bpm": 72}],
    "steps_10min": 45,
    "location": {"lat": 35.6478, "lng": 128.7341, "accuracy": 10}
  }'
```

### 이상 심박 전송 (감지 테스트)
```bash
curl -X POST http://localhost:3000/health \
  -H "Content-Type: application/json" \
  -d '{
    "device_id": "fit3-001",
    "user_id": "위에서_받은_UUID",
    "timestamp": "2026-05-25T15:05:00+09:00",
    "heart_rate": [{"t": "2026-05-25T15:05:00+09:00", "bpm": 130}],
    "steps_10min": 0,
    "location": {"lat": 35.6478, "lng": 128.7341, "accuracy": 10}
  }'
```

  $ curl ...

## 자주 발생하는 에러

### `Error: listen EADDRINUSE: address already in use :::3000`

포트가 이미 사용 중. 기존 프로세스 종료 후 재실행:

```bash
# 3000 포트 쓰는 프로세스 죽이기
lsof -ti:3000 | xargs kill -9

# 다시 실행
npx ts-node src/index.ts
```

### `Error: supabaseUrl is required`

`.env` 파일이 없거나 값이 비어있음. `cp .env.example .env` 후 값 채우기.

### `ENOTFOUND xxxxxxx.supabase.co`

인터넷 연결 확인. 또는 SUPABASE_URL 오타 확인.

### `DB error` (사용자 등록 시)

Supabase SQL Editor에서 `schema.sql` 실행했는지 확인.
테이블이 없으면 이 에러 발생.

## API 엔드포인트 목록 (로컬 테스트용)

| 메서드 | URL | 용도 |
|---|---|---|
| GET | http://localhost:3000/ | 서버 상태 확인 |
| POST | http://localhost:3000/users/register | 농업인 등록 |
| GET | http://localhost:3000/users | 농업인 목록 (대시보드) |
| GET | http://localhost:3000/users/:id | 농업인 상세 |
| GET | http://localhost:3000/users/:id/baseline | 기준선 조회 |
| POST | http://localhost:3000/health | 헬스 데이터 전송 |
| POST | http://localhost:3000/alert | 디바이스 이벤트 |
| GET | http://localhost:3000/alert | 알림 이력 |
| GET | http://localhost:3000/alert/:id | 알림 상세 |
| PATCH | http://localhost:3000/alert/:id | 알림 상태 변경 |
