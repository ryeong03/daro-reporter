## 🟢 임현진 — 헬스 데이터 (Android + Health Connect)

### 선행 조건

- 백엔드 API URL 확정 (Railway 배포 후 전달받기)
- `POST /health` 페이로드 스키마 확인 (`docs/기능명세서.md` 참고)

### 할 일

- **Health Connect 연동**: 심박수 10초 주기 읽기
- **Health Connect 연동**: 걸음수 10분 누적 읽기
- **Retrofit 통신**: `POST /health` API 호출 구현
  ```json
  {
    "device_id": "string",
    "user_id": "UUID",
    "timestamp": "ISO8601",
    "heart_rate": [{"t": "ISO8601", "bpm": 78}],
    "steps_10min": 42,
    "location": {"lat": 35.64, "lng": 128.73, "accuracy": 15}
  }
  ```
- **ForegroundService**: 앱 종료해도 백그라운드 수집 유지
- **GPS 위치 수집**: 정상 시 5분 주기, 이상 감지 응답 시 30초 주기
- **낙상 감지 이벤트 수신**: Galaxy Fit 낙상 이벤트 → `POST /alert` 전송
  ```json
  {
    "device_id": "string",
    "user_id": "UUID",
    "type": "fall_detected",
    "timestamp": "ISO8601",
    "location": {"lat": 35.64, "lng": 128.73}
  }
  ```
- **초기 설정 화면**: 사용자 등록 (이름, 전화번호, 보호자) → `POST /users/register`
- **앱 최초 설치 시**: 과거 7일 심박 데이터 → `heart_rate_history` 배열로 전달
- **상태 표시 화면**: 현재 연결 상태(시계 연결됨/끊김), 마지막 전송 시각

### 테스트

- Health Connect 테스트 앱에서 더미 심박 입력 → 서버 수신 확인
- ForegroundService 알림 표시 + 앱 종료 후에도 동작 확인
- GPS 권한 요청 + 실제 위치 전송 확인

## 🟡 Joonghyuk Kim — 외부 API (카카오맵 + 기상청)

### 선행 조건

- 카카오 개발자 계정 + REST API 키 발급
- 공공데이터포털 가입 + 기상청 API 활용 신청

### 할 일 (백엔드 모듈은 이미 작성됨, 테스트+보완 필요)

- **기상청 API 키 발급** (data.go.kr → 기상청 초단기실황 활용신청)
- **카카오맵 API 키 발급** (developers.kakao.com → REST API 키)
- **기상청 API 테스트**: 청도 다로리 좌표(35.6478, 128.7341)로 호출 확인
  - 파일: `backend/src/external/weather.ts`
  - 함수: `checkWeather(35.6478, 128.7341)`
  - 예상 응답: `{ temperature: 32, humidity: 65, feelsLike: 38.5 }`
- **격자 좌표 변환 검증**: 청도 다로리가 올바른 nx, ny로 변환되는지 확인
- **카카오맵 역지오코딩 테스트**: GPS → "경북 청도군 각남면 다로리" 변환 확인
  - 파일: `backend/src/external/kakao-map.ts`
  - 함수: `coordToAddress(35.6478, 128.7341)`
- **체감온도 계산 로직 검증**: 33°C 이상일 때 관찰 윈도우 단축 동작 확인
- **대시보드 지도 연동** (Phase 2): 카카오맵 JavaScript SDK로 농업인 위치 표시

### 산출물

- API 키를 `.env`에 추가한 뒤 백엔드 팀(세령)에게 전달
- 기상청 격자 좌표 변환 정확도 보고 (청도 다로리 + 주변 2~3개 지점)

## 🟣 박채린 — QA + 고도화

### MVP 테스트 (배포 완료 후)

- **사용자 등록 플로우 테스트** (curl 또는 Postman)
- **헬스 데이터 전송 → 감지 → 알림 E2E 테스트**
  - 정상 심박 전송 → 상태 normal 확인
  - 심박 50% 상승 + 5분 지속 + 걸음수 0 + 2분 미회복 → AI콜 트리거 확인
  - 심박 30% 급락 + 2분 미회복 → AI콜 트리거 확인
- **AI콜 시나리오 테스트**
  - safe 응답 → 정상 종료 확인
  - emergency 응답 → 보건소 알림 확인
  - 무응답 → 재발신 → 무응답 → 보건소 알림 확인
- **대시보드 화면 QA**
  - 농업인 목록 표시 확인
  - 응급 배너 노출 확인
  - 알림 이력 + 오탐 처리 확인
  - 개인 상세 페이지 정보 정확성
- **엣지 케이스 확인**
  - 동일 사용자 연속 알림 시 중복 발생 여부
  - 기준선 75bpm 임시값에서 감지 정확도

### 문서화

- 테스트 결과 리포트 (정상/비정상 케이스 목록)
