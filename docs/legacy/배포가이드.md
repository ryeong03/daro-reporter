## 1. 백엔드 배포 (Railway)

### Railway란?
- GitHub 연동만 하면 자동 배포되는 서버 호스팅
- Node.js 자동 감지, 환경변수 GUI 설정
- 무료 플랜: 월 $5 크레딧 (MVP 테스트 충분)

### 배포 순서

#### 1) Railway 가입 + 프로젝트 생성
1. https://railway.app 접속 → GitHub 계정으로 로그인
2. "New Project" → "Deploy from GitHub repo" 선택
3. `ryeong03/daro-reporter` 레포 연결
4. Root Directory를 `backend`으로 설정

#### 2) 환경변수 설정
Railway 대시보드 → Variables 탭에서 `.env.example`의 모든 키 입력:

```
PORT=3000
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-key
TWILIO_ACCOUNT_SID=ACxxxxxxxx
TWILIO_AUTH_TOKEN=your-token
TWILIO_PHONE_NUMBER=+8210xxxxxxxx
HEALTH_CENTER_PHONE=+8254xxxxxxxx
CLOVA_SPEECH_INVOKE_URL=https://clovaspeech-gw.ncloud.com/recog/v1/stt
CLOVA_SPEECH_SECRET=your-secret
ANTHROPIC_API_KEY=sk-ant-xxxxxxxx
SLACK_WEBHOOK_URL=https://hooks.slack.com/services/xxx/xxx/xxx
KMA_API_KEY=your-kma-key
KAKAO_MAP_API_KEY=your-kakao-key
```

#### 3) 빌드 설정
Railway가 자동 감지하지만 수동으로 설정할 경우:

- **Build Command**: `npm install && npm run build`
- **Start Command**: `npm start`
- **Watch Path**: `/backend`

#### 4) 배포 완료 후
- Railway가 `https://your-app.up.railway.app` 도메인 제공
- 이 URL을 `BASE_URL` 환경변수에 설정 (Twilio 콜백용)
- Twilio 콘솔에서 webhook URL도 이 도메인으로 변경

## 2. 프론트엔드(대시보드) 배포 (Vercel)

### Vercel이란?
- React/Next.js 특화 정적 호스팅
- GitHub push만 하면 자동 빌드+배포
- 무료 플랜으로 충분

### 배포 순서

#### 1) Vercel 가입 + 프로젝트 생성
1. https://vercel.com 접속 → GitHub 계정으로 로그인
2. "Add New Project" → `ryeong03/daro-reporter` 연결
3. **Framework Preset**: Create React App
4. **Root Directory**: `dashboard`

#### 2) 환경변수 설정
Vercel 대시보드 → Settings → Environment Variables:

```
REACT_APP_API_URL=https://your-app.up.railway.app
```

> Railway 배포 후 받은 URL을 여기에 입력

#### 3) 빌드 설정
- **Build Command**: `npm run build`
- **Output Directory**: `build`
- **Install Command**: `npm install`

#### 4) 배포 완료 후
- Vercel이 `https://your-dashboard.vercel.app` 도메인 제공
- 이 URL로 PC 브라우저에서 관리자 대시보드 접속

## 3. 데이터베이스 (Supabase)

이미 사용 중. 별도 배포 필요 없음.

1. https://supabase.com → 프로젝트 생성 (이미 완료 시 스킵)
2. SQL Editor에서 `backend/src/db/schema.sql` 실행
3. Settings → API에서 URL + Service Role Key 복사 → Railway 환경변수에 입력

---

## 4. CORS 설정 (중요!)

배포 후 프론트(Vercel) → 백엔드(Railway) 호출 시 CORS 에러 방지:

백엔드 `src/index.ts`의 cors 설정을 변경 필요:

```typescript
app.use(cors({
  origin: [
    'http://localhost:3001',
    'https://your-dashboard.vercel.app'  // 배포 후 실제 도메인
  ]
}));
```

### curl로 사용자 등록
```bash

  -H "Content-Type: application/json" \
  -d '{
    "name": "김할머니",
    "phone": "01011112222",
    "device_id": "fit3-test-001",
    "gender": "female",
    "birth_date": "1945-03-15",
    "guardians": [{"name": "김보호자", "phone": "01033334444", "relation": "아들"}]
```
