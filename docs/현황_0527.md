## 5/27 변경사항

### 백엔드 NestJS 마이그레이션 완료

> 멘토 피드백: "Express(Node.js) 대신 NestJS 쓰라"

| 항목 | 이전 | 이후 |
|---|---|---|
| 프레임워크 | Express 4.21 | **NestJS 11** |
| 구조 | `api/*.ts` 라우트 핸들러 단일 레이어 | 모듈/컨트롤러/서비스 3계층 분리 |
| DI | 없음 (import 직접 참조) | NestJS IoC 컨테이너 기반 의존성 주입 |
| 설정 관리 | `dotenv` 수동 로드 | `@nestjs/config` ConfigModule |
| 개발 서버 | nodemon | `nest start --watch` |

**모듈 구조:**
```
src/
├── app.module.ts              (루트 모듈)
├── main.ts                    (엔트리포인트)
├── database/                  (Supabase — Global)
├── state/                     (상태 머신 — Global)
├── external/                  (기상청, 카카오맵 — Global)
├── detection/                 (감지 엔진 + 기준선)
├── ai-call/                   (Twilio 발신, STT, Claude 분류 — Global)
├── notify/                    (SMS, Slack, 긴급알림 — Global)
├── health/                    (헬스 데이터 수신 API)
├── alert/                     (알림 CRUD API)
├── users/                     (사용자 등록/조회 API)
├── twilio-webhook/            (Twilio 콜백 처리)
└── guardian/                  (보호자 인증/조회 API — 5/27 추가)
```

**API 엔드포인트 변경 없음** — 프론트엔드/모바일 수정 불필요.

### Docker 빌드 오류 수정

**증상**: Railway 배포 시 컨테이너가 `Cannot find module '/app/dist/main'`으로 반복 크래시.

**원인**: TypeScript의 `incremental: true` 설정으로 생성되는 `*.tsbuildinfo` 캐시 파일이 `.dockerignore`에 없어 Docker 이미지에 복사됨. `nest build` 시 `deleteOutDir: true`로 `dist/`를 삭제한 뒤, 캐시가 "변경 없음"으로 판단해 아무 파일도 생성하지 않는 문제.

**수정 내용**:

| 파일 | 변경 |
|---|---|
| `backend/.dockerignore` | `*.tsbuildinfo` 패턴 추가 |
| `backend/Dockerfile` | 빌드 전 `rm -f *.tsbuildinfo` 추가 |
| `backend/tsconfig.build.json` | NestJS 표준 빌드 설정 파일 신규 생성 |
| `.gitignore` | `*.tsbuildinfo` 패턴 추가 |

### 🤔 논의 필요: AI콜 이후 프론트 역할이 있는가?

#### 현재 구현된 AI콜 흐름 (전부 백엔드에서 처리됨)

```
이상 감지 → Twilio 전화 발신 → 어르신 전화 받음
→ TwiML 음성 안내 ("괜찮으시면 말씀해주세요")
→ 녹음 → Clova STT → Claude 분류
→ safe: 종료 / emergency: 보건소 알림 / unclear: 재발신
```

> 이 플로우는 **전부 서버 + Twilio가 처리**함.
> 어르신 폰에서는 그냥 일반 전화가 오는 것처럼 보임.
> 앱 화면이 뜨거나 하지 않음.

#### 그래서 고민되는 것들

| 질문 | 현재 상황 | 고민 포인트 |
|---|---|---|
| "AI콜 프론트"가 필요한가? | Twilio 전화는 일반 통화로 옴 → 앱 UI 불필요 | 프론트 구현할 게 없을 수도 있음 |
| 전화 받으면 앱으로 넘어가게 해야 하나? | 현재는 안 넘어감 (일반 전화) | 딥링크로 가능은 하지만, 어르신이 앱 조작하는 게 현실적인지? |
| 대시보드에 AI콜 상태 표시는? | call_logs 테이블에 기록은 됨 | 관리자가 실시간으로 "지금 통화 중" 같은 걸 봐야 하나? |
| 어르신 앱에 "방금 AI콜 왔었음" 알림? | 현재 없음 | 사후 기록으로 보여주는 건 있으면 좋을 수도 |

#### 내 생각 (세령 의견)

1. Twilio 전화는 일반 통화임 → 앱 UI가 끼어들 틈이 없음
2. 어르신은 고령자 → 전화 받고 말하는 것만으로도 충분, 앱 화면까지 보게 하는 건 과함
3. 실제로 필요한 건 **대시보드에서 관리자가 AI콜 이력을 보는 것** 정도

만약 프론트 작업을 넣는다면:
- 대시보드에 "AI콜 이력 페이지" 추가 (call_logs 조회)
- 어르신 앱 홈에 "최근 안부 확인 기록" 카드 정도

→ **수요일 멘토 미팅에서 같이 물어보자**

#### 💡 고도화 아이디어

- AI콜 음성을 **가족(손녀 등) 목소리로 변환**해서 전화하는 방향도 고려 중
  - 어르신이 낯선 AI 목소리보다 익숙한 가족 목소리에 더 잘 반응할 수 있음
  - 보호자 등록 시 음성 샘플 녹음 → TTS 음성 합성(Voice Cloning) 활용
  - 기술적으로는 ElevenLabs, NAVER Clova Voice 등 활용 가능
- 아직 구체적 계획은 아니고, 방향성 차원에서 검토 중

### PWA vs Android 네이티브

> **어르신 앱은 반드시 Android 네이티브여야 한다.**

| 기능 | 웹 (PWA) | Android 네이티브 |
|---|---|---|
| Health Connect (심박/걸음수) | **불가능** | 가능 |
| GPS 백그라운드 수집 | 제한적 (브라우저 닫으면 끊김) | 가능 (ForegroundService) |
| Foreground Service (상시 구동) | **불가능** | 가능 |

Health Connect API = Android SDK 전용. 웹 브라우저에는 API 자체가 없음.
Galaxy Fit 3 → Health Connect → 우리 앱 경로는 네이티브 앱에서만 동작.

- **어르신 앱** → Android 네이티브 필수 (PWA 대체 불가)
- **관리자 대시보드** → 웹 + PWA 적용 (데이터 조회 전용)
- **모바일 웹앱** (`daro-reporter-mobile.vercel.app`) → 데모/시연용

→ `android/` 빌드 환경 구축은 빠질 수 없는 핵심 작업. 현진 빨리 착수 필요.
