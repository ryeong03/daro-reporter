# 바이브코딩 환경 준비 (비개발자용)

> Git 클론만으로는 실행이 안 됨. **`npm install`로 `node_modules`를 받은 뒤** Cursor에서 작업 시작.

로컬에서 서버까지 띄우려면 → [로컬개발_시작가이드.md](./로컬개발_시작가이드.md)

---

## 클론했는데 `node_modules`가 없는 이유

| 이름 | Git에 있음? | 설명 |
|---|---|---|
| 소스 코드 (`src/`, 화면 파일 등) | ✅ | `git clone`으로 받음 |
| **`node_modules/`** | ❌ (`.gitignore`) | 라이브러리 폴더. **직접 생성 필요** |
| **`.env`** | ❌ | API 키. **팀에서 받아서 직접 생성** |

`node_modules`는 용량이 커서 Git에 안 올림.  
**작업할 폴더에서 `npm install` 한 번** → 그 폴더 안에 `node_modules`가 생김.

```
daro-reporter/
├── backend/node_modules/     ← cd backend && npm install
├── dashboard/node_modules/   ← cd dashboard && npm install
└── mobile/node_modules/      ← cd mobile && npm install
```

루트(`daro-reporter/`)에는 `node_modules` 없음. **폴더마다 따로 설치.**

---

## 0. 한 번만 설치

| 프로그램 | 확인 | 없으면 |
|---|---|---|
| **Node.js** v18+ | 터미널: `node -v` | https://nodejs.org (LTS) |
| **Git** | `git --version` | https://git-scm.com |
| **Cursor** | 앱 실행 | https://cursor.com |

Node 설치 후 **Cursor 완전 종료 → 다시 실행** (터미널이 Node 인식하게).

---

## 1. 프로젝트 받기

**Cursor**

1. Clone 또는 **File → Open Folder**
2. `https://github.com/ryeong03/daro-reporter.git`
3. 열린 폴더 이름이 `daro-reporter`인지 확인

**터미널**

```bash
git clone https://github.com/ryeong03/daro-reporter.git
cd daro-reporter
```

---

## 2. Cursor 터미널 열기

- **Ctrl + `** (Mac도 동일한 경우 많음)
- 또는 상단 **Terminal → New Terminal**
- 아래에 검은/흰 창 + `daro-reporter` 경로 보이면 OK

---

## 3. `node_modules` 받기 (`npm install`)

**내가 손댈 폴더만** 설치하면 됨.

| 담당 | 폴더 | 명령 |
|---|---|---|
| PC 대시보드 | `dashboard/` | `cd dashboard` → `npm install` |
| 백엔드 API | `backend/` | `cd backend` → `npm install` |
| 모바일 웹 | `mobile/` | `cd mobile` → `npm install` |

### 예: 대시보드만

```bash
cd dashboard
npm install
```

끝나면 `dashboard/node_modules` 생성 (파일 수천 개 — 정상).  
**첫 설치 1~5분**, 인터넷 필요.

### 여러 폴더 한 번에 (프로젝트 루트에서)

```bash
cd dashboard && npm install && cd ..
cd backend && npm install && cd ..
cd mobile && npm install && cd ..
```

---

## 4. `.env` (팀에서 받기)

Git에 없음. 비어 있으면 서버·지도 연동 안 됨.

| 폴더 | 만들기 |
|---|---|
| `backend/` | `cp .env.example .env` → Supabase 키 등 **세령에게 요청** |
| `dashboard/` | `cp .env.example .env` → API URL, 카카오맵 키 **세령에게 요청** |

UI만 수정할 때 (배포된 API 쓰기):

```env
REACT_APP_API_URL=https://daro-reporter-production.up.railway.app
REACT_APP_KAKAO_MAP_KEY=(팀에서 받은 JavaScript 키)
```

---

## 5. 잘 됐는지 확인

### 대시보드

```bash
cd dashboard
npm start
```

브라우저에 화면 뜨면 OK. 끄기: 터미널 `Ctrl + C`.

### 백엔드 (로컬 API까지)

터미널 2개:

```bash
# 터미널 1
cd backend
npm run start:dev

# 터미널 2
cd dashboard
npm start
```

http://localhost:3000 → JSON 응답 나오면 백엔드 OK.

## 자주 나는 문제

| 증상 | 해결 |
|---|---|
| `npm: command not found` | Node 설치 후 Cursor 재시작 |
| `Cannot find module ...` | 그 폴더에서 `npm install` 안 함 → `cd` 확인 후 재실행 |
| `.env` 없다는 에러 | `cp .env.example .env` + 팀에 키 요청 |
| `EADDRINUSE` (포트 사용 중) | 터미널 `Ctrl + C`로 서버 끄고 다시 `npm start` |
