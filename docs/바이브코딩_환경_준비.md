# 바이브코딩 환경 준비 (비개발자용)

> Git 클론만으로는 코드가 바로 안 돌아감. 이 문서 순서대로 하면 Cursor에서 작업 시작 가능.

---

## 왜 클론만으로는 부족한가

| 이름 | Git에 올라감? | 역할 |
|---|---|---|
| 소스 코드 (`src/`, `*.tsx` 등) | ✅ 올라감 | 우리가 수정하는 파일 |
| **`node_modules/`** | ❌ 안 올라감 (`.gitignore`) | 라이브러리 묶음. **직접 받아야 함** |
| **`.env`** | ❌ 안 올라감 (비밀키) | API 키 등. **팀에서 받아서 직접 만들기** |

`node_modules`는 용량이 커서 Git에 안 넣음. 대신 **`npm install`** 한 번이면 프로젝트마다 자동 생성됨.

---

## 0. 미리 설치할 것 (한 번만)

| 프로그램 | 확인 방법 | 없으면 |
|---|---|---|
| **Node.js** (v18 이상) | 터미널에 `node -v` 입력 → 버전 나오면 OK | https://nodejs.org (LTS 설치) |
| **Git** | `git --version` | https://git-scm.com |
| **Cursor** | 앱 실행 가능하면 OK | https://cursor.com |

Node 설치 후 **Cursor를 완전히 껐다 켜기** (터미널이 Node를 인식하게).

---

## 1. 프로젝트 받기

### Cursor에서

1. **File → Open Folder** (또는 Clone)
2. 저장소 주소:
   ```
   https://github.com/ryeong03/daro-reporter.git
   ```
3. 폴더 선택 후 열기 → 프로젝트 루트가 `daro-reporter` 여야 함

### 터미널에서 (선택)

```bash
git clone https://github.com/ryeong03/daro-reporter.git
cd daro-reporter
```

---

## 2. Cursor 터미널 열기

- 단축키: **Ctrl + `** (Mac: **Control + `** 또는 상단 메뉴 **Terminal → New Terminal**)
- 아래쪽에 검은/흰 창이 뜨면 성공
- 프롬프트 앞에 `daro-reporter` 같은 경로가 보이면 루트 폴더에서 실행 중

---

## 3. `node_modules` 받기 — `npm install`

**작업할 폴더만** 설치하면 됨. 전부 할 필요 없음.

### 담당별로

| 담당 | 폴더 | 명령 |
|---|---|---|
| 대시보드 (PC 화면) | `dashboard/` | 아래 ① |
| 백엔드 API | `backend/` | 아래 ② |
| 모바일 웹 | `mobile/` | 아래 ③ |

### ① 대시보드

```bash
cd dashboard
npm install
```

끝나면 `dashboard/node_modules` 폴더가 생김 (수천 개 파일 — 정상).

### ② 백엔드

```bash
cd backend
npm install
```

### ③ 모바일 (웹만 볼 때)

```bash
cd mobile
npm install
```

### 한 번에 여러 개 (루트에서)

```bash
cd dashboard && npm install && cd ..
cd backend && npm install && cd ..
cd mobile && npm install && cd ..
```

> **첫 `npm install`은 1~5분** 걸릴 수 있음. 인터넷 필요.

---

## 4. 환경변수 `.env` (팀에서 받기)

`.env`도 Git에 없음. 비어 있으면 서버/지도가 안 됨.

| 폴더 | 만들 방법 | 필요한 값 |
|---|---|---|
| `backend/` | `cp .env.example .env` 후 편집 | Supabase 등 — **세령에게 요청** |
| `dashboard/` | `cp .env.example .env` 후 편집 | API URL, 카카오맵 키 — **세령에게 요청** |

로컬만 UI 수정할 때는 `dashboard/.env`에 아래만 있어도 됨 (백엔드는 배포 URL 사용):

```env
REACT_APP_API_URL=https://daro-reporter-production.up.railway.app
REACT_APP_KAKAO_MAP_KEY=(팀에서 받은 JavaScript 키)
```

---

## 5. 잘 됐는지 확인

### 대시보드만 할 때

```bash
cd dashboard
npm start
```

브라우저에 페이지 열리면 OK. 종료: 터미널에서 `Ctrl + C`.

### 백엔드까지 로컬에서 돌릴 때

터미널 **두 개** 필요:

```bash
# 터미널 1
cd backend
npm run start:dev

# 터미널 2 (새 탭: Cmd+Shift+` 또는 터미널 + 버튼)
cd dashboard
npm start
```

브라우저 `http://localhost:3000` → `{"status":"ok",...}` 비슷한 JSON이면 백엔드 OK.

## 6. Cursor에 넣을 첫 메시지 (복붙용)

환경 준비 끝난 뒤, 채팅에 이렇게 붙여넣기:

```
이 프로젝트는 daro-reporter (Hero)야.
나는 [대시보드 / 백엔드 / 모바일] 파트 작업 중이고,
npm install은 [dashboard / backend / mobile] 폴더에서 이미 했어.
docs/바이브코딩_환경_준비.md 를 읽고 내 작업 폴더 기준으로 도와줘.
```

예시 (대시보드):

```
대시보드(dashboard/) UI 수정할 거야.
npm install 완료했고 REACT_APP_API_URL은 Railway 배포 URL 쓰고 있어.
UsersPage.tsx 에서 목록 카드 레이아웃만 바꿔줘.
```

## 자주 나는 상황

### `npm: command not found`

→ Node.js 미설치 또는 Cursor 재시작 전. Node 설치 후 Cursor 완전 종료 후 다시 실행.

### `Cannot find module ...` / 빨간 줄 잔뜩

→ 해당 폴더에서 `npm install` 안 함. `cd dashboard` (또는 backend/mobile) 후 `npm install` 다시.

### `ENOENT: no such file or directory, open '.env'`

→ `.env.example`을 복사해서 `.env` 만들기. 값은 팀에 문의.

### `npm install` 중 에러 (permission / EACCES)

→ Mac이면 보통 `sudo` 없이 그냥 재시도. 계속 실패하면 팀 Slack에 스크린샷.

### 포트 이미 사용 중 (`EADDRINUSE`)

→ 예전에 켜 둔 서버가 남음. 터미널에서 `Ctrl + C`로 끄거나 Cursor 터미널 탭 전부 닫고 다시 `npm start`.

## 폴더 구조 한 줄 요약

```
daro-reporter/
├── backend/      ← API 서버 (npm install 필수)
├── dashboard/    ← PC 관리 화면 (npm install 필수)
├── mobile/       ← 어르신 앱 / 웹 (npm install 필수)
├── docs/         ← 문서 (이 파일 포함)
└── node_modules  ← 루트에는 없음. 각 하위 폴더에만 생김
```
